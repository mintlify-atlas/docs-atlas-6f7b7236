Customer Management handles customer registration, lookup, and provides customers with a comprehensive dashboard to track their shipments and transactions.

## Customer Registration

The system supports both new customer registration and lookup of existing customers:

### Registration Process

<Steps>
  <Step title="Enter Customer Number">
    Input the customer's unique identification number.
    
    ```php
    // CustomerController.php:124-128
    $this->validate($request, [
        'CustNumber' => ['required'],
        'CustAddress' => ['required'],
        'LFrom' => ['required'],
        'ULTo' => ['required']
    ]);
    ```
  </Step>
  
  <Step title="Check Existing Records">
    System automatically checks if customer exists:
    
    ```php
    // CustomerController.php:138-151
    if (customer::where('CustNumber', request('CustNumber'))->exists()){
        $_Customer = customer::where('CustNumber', request('CustNumber'))->get();
        
        // Update name if changed
        if($_Customer[0]->CustName != request('CustName')){
            Customer::where('CustNumber', request('CustNumber'))->update([
                'CustName' => request('CustName')
            ]);
        }
        $Cust = customer::all('id','CustNumber');
    } else {
        $NewCustomer->save();
        $Cust = customer::all('id','CustNumber');
    }
    ```
  </Step>
  
  <Step title="Create or Update Profile">
    - **New Customer**: Creates new record with name and number
    - **Existing Customer**: Updates name if changed
    - Associates customer with new package
  </Step>
</Steps>

<Note>
The system automatically updates customer names if they differ from previous records, ensuring information stays current.
</Note>

## Customer Dashboard

The customer dashboard provides a comprehensive view of shipment activity:

### Dashboard Overview

```php
// CustomerController.php:51-72
public function dashboard()
{
    $datetime = date('Y-m-d H:i:s');
    $LastPack = package::with('VesselName')
        ->where([['customer_id',auth()->user()->custid]])
        ->orderByDesc('id')
        ->First();
    
    $transaction = package::where('customer_id',auth()->user()->custid)
        ->orderByDesc('id')
        ->get();
    
    $TransColl = package::where([['customer_id',auth()->user()->custid],
                                  ['status','COLLECTED']])->get();
    
    $TransShipped = package::where([['customer_id',auth()->user()->custid],
                                     ['status','!=','COLLECTED']])->get();
    
    $transaction->TotalShipments = count($transaction);
    $transaction->Collected = count($TransColl);
    $transaction->Shipped = count($TransShipped);
}
```

### Departure Time Tracking

For packages not yet collected, the dashboard shows vessel departure information:

```php
// CustomerController.php:61-68
if($LastPack){
```

## Settlement Tracking

The settlement view shows payment status for all transactions:

```php
// CustomerController.php:83-105
public function settlement()
{
    $transaction = package::where('customer_id',auth()->user()->custid)
        ->orderByDesc('id')
        ->get();
    $items = shipment::orderByDesc('packages_id')->get();
    $receivables = receivables::orderByDesc('packID')->get();
    
    foreach($transaction as $trans){
        $total = 0;
        
        // Check payment status
        foreach($receivables as $receiv){
            if($trans->id == $receiv->packID){
                $trans->paymentStatus = "PAID";
                break;
            }
            $trans->paymentStatus = "CREDIT";
        }
        
        // Calculate total
        foreach($items as $item){
            if($trans->id == $item->packages_id){
                $total += $item->qty * $item->unit_price;
            }
        }
        $trans->total = $total;
    }
    
    return view('customer.settlement',['transaction'=>$transaction]);
}
```

### Payment Status

Each transaction shows:
- **PAID**: Payment has been recorded in receivables
- **CREDIT**: No payment record found (POD or pending)

### Total Calculation

The system calculates totals by:
1. Finding all shipment items for the package
2. Multiplying quantity × unit price for each item
3. Summing all item totals

## Guest Customers

The system supports anonymous shipments through a special guest account:

```php
// ShipmentController.php:124-136
$NewCustomer = new customer([
    'CustNumber' => 0,
    'CustName' => "Guest"
]);

if (customer::where('CustNumber', 0)->exists()){
    $Cust = customer::all('id','CustNumber');
} else {
    $NewCustomer->save();
    $Cust = customer::all('id','CustNumber');
}
```

<Warning>
Guest customers use `CustNumber = 0` and cannot access the customer dashboard. These are for walk-in or one-time shipments.
</Warning>

## Best Practices

1. **Accurate Information**: Ensure customer numbers are correct for proper record linkage
2. **Name Updates**: The system auto-updates names, but verify for important changes
3. **Address Verification**: Double-check delivery addresses during package creation
4. **Dashboard Training**: Educate customers on using the dashboard for self-service tracking

## Common Scenarios

### Scenario 1: Regular Customer Returns

1. Operator enters customer number during shipment creation
2. System retrieves existing customer record
3. Auto-fills customer name
4. Operator verifies or updates name if needed
5. Proceeds with package creation

### Scenario 2: New Customer Registration

1. Customer arrives with new shipment
2. Operator enters new customer number and name
3. System creates customer record
4. Links customer to package
5. Customer can now access dashboard with credentials

### Scenario 3: Customer Checks Status Online

1. Customer logs into dashboard
2. Views last package status
3. Sees vessel has departed (isDepatured = true)
4. Checks settlement to confirm payment status
5. Views transaction history for past shipments
