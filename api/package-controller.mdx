## Overview

The `PackageController` handles the finalization of packages after shipment items have been added. It manages payment processing (cash and transfer), package status updates, and cancellation of packages with associated items. It also includes a Viber bot integration for customer notifications.

**Location:** `app/Http/Controllers/PackageController.php`

**Related Models:**
- [Package](/models/package)
- [Shipment](/models/shipment)
- [Receivables](/models/receivables)

- [Category](/models/category)

### store()

Processes package finalization with payment or cancels the package.

<ParamField path="request" type="Request" required>
  The HTTP request object with package and payment data
  
  <ParamField path="SubmitType" type="string" required>
    Action type: "SAVE" or "CANCEL"
  </ParamField>
  
  <ParamField path="packageID" type="integer" required>
    Package ID to finalize or cancel
  </ParamField>
  
  <ParamField path="payOption" type="string">
    Payment option: "POD" (Pay on Delivery) or "NOW"
  </ParamField>
  
  <ParamField path="payType" type="string">
    Payment type: "CASH" or "TRANSFER"
  </ParamField>
  
  <ParamField path="shipmentTotal" type="decimal">
    Total amount for the shipment
  </ParamField>
  
  <ParamField path="paySlip" type="file">
    Payment slip image (required for TRANSFER payment)
  </ParamField>
</ParamField>

**Returns:** 
- Success: Redirect to `/confirm`
- Error: Redirect to `/create?packid={id}` with error message

**Route:** `POST /confirm`

**Business Logic:**

**For "SAVE" action:**
1. Validates that package has items
2. If payment option is "POD", skips payment processing
3. If payment option is "NOW":
   - For "CASH": Creates receivable record without slip
   - For "TRANSFER": Validates slip attachment, uploads image, creates receivable
4. Updates package status to "LOADED"
5. Redirects to confirmation page

**For "CANCEL" action:**
1. Deletes all shipment items associated with package
2. Deletes the package record
3. Clears session data
4. Redirects to create page

**Image Handling (Payment Slips):**
- Filename format: `{timestamp}_{packageID}.{extension}`
- Storage location: `public/img/`

**Code Example:**
```php
public function store(Request $request)
{
    if ($request->SubmitType == "SAVE"){
        if(!shipment::where('packages_id', request('packageID'))->exists()) {
            return redirect('/create')->with('status_error', 'No item added ');
        } 
        if($request->payOption != "POD"){
            if($request->payType == "CASH"){
                $Newreceivables = new receivables ([
                    'packID' => request('packageID'),
                    'paymentType' => Request('payType'),
                    'payslip' => '',
                    'total' => $request->shipmentTotal 
                ]);
                $Newreceivables->save();
            }elseif($request->payType == "TRANSFER"){
                if($request->paySlip != null){
                    $newPath = time() . "_" . $request['packageID'] . "." . $request['paySlip']->extension();
                    request('paySlip')->move(public_path("img"), $newPath);

                    $Newreceivables = new receivables ([
                        'packID' => request('packageID'),
                        'paymentType' => Request('payType'),
                        'payslip' => $newPath,
                        'total' => $request->shipmentTotal 
                    ]);
                    $Newreceivables->save();
                }else{
                    return redirect('/create?packid='.$request->packageID)->with('status_error','Please Attach the slip');
                }
            }
        }

        package::where('id', $request->packageID)->update([
            'status' => "LOADED"
        ]);
        return redirect('/confirm');
    }else if($request->SubmitType == "CANCEL") {
        if(shipment::where('packages_id', Session::get('newpackage')->id)->exists()){
            $deleteItem = shipment::where('packages_id', Session::get('newpackage')->id);
            $deleteItem->delete();
        }
        if(Package::where('id', Session::get('newpackage')->id)->exists()){
            $deleteItem = package::find(Session::get('newpackage')->id);
            $deleteItem->delete();
        }
        Session::forget('newpackage');
        Session::forget('NewCustomer');
        return redirect('/create');
    }else{
        return redirect('/create');
    }
}
```

### Bot()

Handles Viber bot webhook events for customer notifications.

<ParamField path="_input" type="mixed" required>
  Input parameter (currently unused)
</ParamField>

**Returns:** JSON webhook response or void

**Route:** Custom webhook endpoint

**Business Logic:**
1. Receives Viber webhook events
2. For 'webhook' event: Returns status confirmation
3. For message events: Processes and responds to messages
4. Includes helper function `sendmessage()` for Viber API calls

**Viber Integration Details:**
- Access Token: Hardcoded in method (should be moved to config)
- Supported events: 'webhook', message events
- Message types: text

**Code Example:**

        $MsgSenderID = $input['sender']['id'];

        curl_setopt($ch, CURLOPT_HTTPHEADER, array('Content_Type : application/json'));

        return $result;
    }

## Payment Processing

### Payment Options

1. **POD (Pay on Delivery)**
   - No immediate payment record created
   - Payment will be processed at delivery/collection

2. **NOW (Pay Now)**
   - Cash: Direct payment without slip
   - Transfer: Requires payment slip upload

### Payment Types

- **CASH**: Direct cash payment
- **TRANSFER**: Bank transfer with slip verification

---

## Error Handling

The controller returns these error messages:

- "No item added" - Attempting to save package without items
- "Please Attach the slip" - Missing payment slip for transfer payment
