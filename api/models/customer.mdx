## Overview

The Customer model represents customers in the NaaleMV shipping system. Each customer has a unique phone number, name, and optional Viber token for notifications. Customers can place multiple package orders and track their shipments.

## Database Table

**Table name:** `customers`

## Customer Types

### Regular Customers

- Have a unique `CustNumber` (phone number)
- Can have multiple packages
- Can optionally integrate with Viber for notifications
- Customer names can be updated if they change

### Guest Customers

- Use `CustNumber` = 0
- Use `CustName` = "Guest"
- Allow for anonymous package creation
- Share the same customer record (singleton pattern)

## Notes

- `CustNumber` is used as the primary identifier for customer lookup, not the database ID
- The system checks if a customer exists before creating a new record to prevent duplicates
- Customer names can be updated if the same phone number is used with a different name
- The `ViberToken` field stores the Viber API token for push notifications
- Guest customers (CustNumber = 0) are used for walk-in or anonymous shipments
