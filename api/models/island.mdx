## Overview

The Island model represents islands in the Maldives. Each island has a name, code, and belongs to an atoll. Islands are used to specify delivery destinations and track geographical locations for package routing.

## Database Table

**Table name:** `islands`

## Example Usage

### Creating a New Island

From `ShipmentController.php:68-74`:

```php
$atoll = atoll::where('name', $request->atoll)->first();

$newIslands = new islands([
    'atoll_id' => $atoll->id,
```

## Notes

- Island names are stored as provided, while atoll names are converted to uppercase
- The system automatically creates a new atoll if it doesn't exist when adding an island
- Islands are typically displayed in alphabetical order by name
- The `code` field provides a short abbreviation for the island, useful for compact displays
- Islands are used in the package creation process to specify delivery destinations
- The relationship with atolls helps organize islands by geographical and administrative divisions

## Geographic Context

The Maldives consists of 26 atolls containing approximately 200 inhabited islands. The Island model helps:

- Organize delivery destinations by atoll
- Route packages efficiently
- Display location information to customers
- Support filtering and searching by geographical area

## Related Models

- **Atoll**: Parent administrative division that contains multiple islands
- **Vessel**: Vessels service specific islands as destinations
- **Package**: Packages specify `from` and `to` locations, which can reference islands
