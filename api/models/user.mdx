## Overview

The User model represents all user accounts in NaaleMV including administrators, vessel operators (crew), and customers. It handles authentication via Laravel Fortify and supports role-based access control. This model is defined in `app/Models/User.php` and uses the `users` database table.

## Usage Examples

### Creating a User

From `SettingController.php:59-85`:

```php
// Create a new crew member
User::create([
    'fullname' => $request['Fname'],
    'contact' => $request['contact'],
    'boatid' => auth()->user()->boatid,
    'rank' => "CREW",
    'email' => $request['email'],
    'password' => Hash::make($request['password']),
])->save();
```

### Changing Password

From `SettingController.php:93-111`:

```php
// Verify old password and update with new one
$user = User::find(auth()->user()->id);

if (!Hash::check($request['old_pass'], $user->password)) {
    // Old password doesn't match
    return redirect()->back()->with('status_error', 'Password doesent match');
}

$user->forceFill([
    'password' => Hash::make($request['new_pass']),
])->save();
```

## Relationships

<Expandable title="Vessel">
  **Type**: belongsTo
  
  ```php
  // Each user belongs to a vessel via boatid foreign key
  Schema::table('users', function (Blueprint $table) {
      $table->foreignId('boatid')
            ->constrained('vessels')
            ->nullable();
  });
  ```
  
  Users (except admin) are assigned to a specific vessel. The `boatid` field links to the `vessels` table.
</Expandable>

<Expandable title="Customer">
  **Type**: Has one (implicit)
  
  When a user has rank "customer", they have an associated customer record in the customers table. The customer record uses the user's contact number to link back.
</Expandable>

## Role-Based Access Control

The `rank` field determines what features users can access:

### Admin
- Manage all vessels and users
- Create islands and atolls
- View all packages and shipments
- Access system settings

### CREW (Vessel Operators)
- Create and manage shipments for their vessel
- Process collections
- Manage their vessel's schedule (dathuru)
- Add crew members to their vessel

### Customer
- View their own packages and transactions
- Check vessel schedules
- View settlement information
- Track package status

## Language Support

The `isDhivehi` field stores the user's language preference:

```php
// Determine language for view rendering
Session::get('isDhivehi') ? $lang = "dhi" : $lang = "eng";

```

- `0` (false): English interface
- `1` (true): Dhivehi interface

## Common Queries

### Get All Crew for a Vessel

```php
$users = User::where('boatid', auth()->user()->boatid)
             ->where('rank', '!=', 'customer')
             ->get();
```

### Check User Rank

```php
if (auth()->user()->rank == "customer") {
    return redirect('/customer/dashboard');
}
```

### Get Active Users

```php
$activeUsers = User::where('status', true)->get();
```
