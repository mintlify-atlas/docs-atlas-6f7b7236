## Overview

The Package model represents a shipping package in the NaaleMV system. Each package contains delivery information, status tracking, and is associated with a customer and vessel. Packages serve as containers for multiple shipment items and track the overall delivery process.

## Database Table

**Table name:** `packages`

## Schema Definition

<Expandable title="hasOne: Payment Status (Receivables)">

<Expandable title="hasOne: Vessel Name">

## Package Status Values

- **LOADING**: Package is being prepared and items are being added
- **LOADED**: Package is complete and ready for shipment
- **COLLECTED**: Package has been delivered and collected by the customer

## Notes

- The `vessel_id` field was added in a later migration and is nullable, indicating that packages can be created before being assigned to a vessel
- Packages in 'LOADING' status represent incomplete packages that are still accepting shipment items
- Guest customers can create packages with customer_id = 0
