## Overview

The Shipment model represents individual items within a package. Each shipment record contains details about the goods being transported, including quantity, pricing, category, and an optional image. Shipments belong to packages and are categorized for organization.

## Database Table

**Table name:** `shipments`

- `category_id` has a foreign key constraint to the `categories` table
- `packages_id` has a foreign key constraint to the `packages` table

## Relationships

<Expandable title="belongsTo: Category">
  Each shipment belongs to a category via the `category_id` foreign key. This relationship is used to classify the type of goods being shipped.
</Expandable>

<Expandable title="belongsTo: Package">
  Each shipment belongs to a package via the `packages_id` foreign key. Multiple shipments can belong to a single package.
</Expandable>

## Notes

- Images are stored in the `item_img` directory with filenames in the format: `{timestamp}_pack_id_{packID}.{extension}`
- When deleting a shipment, the associated image file should also be deleted from storage
- The total value of a shipment is calculated as `qty * unit_price`
