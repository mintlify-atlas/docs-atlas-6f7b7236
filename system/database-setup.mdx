## Database Overview

NaaleMV uses MySQL 5.7+ as its primary database system. The application manages shipping packages, vessels, customers, and related operations through a relational database schema.

## Creating the Database

### Standard MySQL Installation

```bash
# Login to MySQL
mysql -u root -p

# Create database
CREATE DATABASE naalemvdb CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# Create user (optional, recommended for production)
CREATE USER 'naalemv_user'@'localhost' IDENTIFIED BY 'your_secure_password';
GRANT ALL PRIVILEGES ON naalemvdb.* TO 'naalemv_user'@'localhost';
FLUSH PRIVILEGES;

# Exit MySQL
EXIT;
```

### Docker Setup

If using the Docker configuration:

```bash
# Start Docker containers
docker-compose up -d

# Database is automatically created as 'naalemvdb'
# Access MySQL in container
docker exec -it naalemvdb mysql -u root

# Verify database
SHOW DATABASES;
```

<Warning>
The Docker setup uses MySQL 5.7.16 on port **3322** (host) mapped to 3306 (container). Update your `.env` file accordingly:
```bash
DB_PORT=3322
```
</Warning>

## Running Migrations

Migrations create the database schema automatically.

### Execute Migrations

```bash
# Run all pending migrations
php artisan migrate

# Run migrations with confirmation prompt
php artisan migrate --step

# Force migrations in production (skips confirmation)
php artisan migrate --force
```

### Rollback Migrations

```bash
# Rollback last batch
php artisan migrate:rollback

# Rollback specific number of migrations
php artisan migrate:rollback --step=5

# Reset all migrations
php artisan migrate:reset

# Refresh database (rollback all and re-migrate)
php artisan migrate:refresh

# Fresh migration (drop all tables and re-migrate)
php artisan migrate:fresh
```

<Warning>
**Destructive Operations**: Commands like `migrate:fresh` and `migrate:refresh` will delete all data. Use with extreme caution in production.
</Warning>

## Database Seeding

### Default Seeder

The database seeder is located at `/database/seeders/DatabaseSeeder.php`. By default, it's configured to use factory-generated test data (commented out).

### Running Seeders

```bash
# Run all seeders
php artisan db:seed

# Run specific seeder
php artisan db:seed --class=DatabaseSeeder

# Refresh database and seed
php artisan migrate:fresh --seed
```

## Backup and Restore

### Backup Database

<CodeGroup>

```bash Standard MySQL
# Backup entire database
mysqldump -u root -p naalemvdb > backup_$(date +%Y%m%d).sql

# Backup with compression
mysqldump -u root -p naalemvdb | gzip > backup_$(date +%Y%m%d).sql.gz

# Backup specific tables
mysqldump -u root -p naalemvdb users vessels packages > backup_core_tables.sql
```

```bash Docker
# Backup from Docker container
docker exec naalemvdb mysqldump -u root naalemvdb > backup_$(date +%Y%m%d).sql

# Backup with compression
docker exec naalemvdb mysqldump -u root naalemvdb | gzip > backup_$(date +%Y%m%d).sql.gz
```

</CodeGroup>

### Restore Database

<CodeGroup>

```bash Standard MySQL
# Restore from backup
mysql -u root -p naalemvdb < backup_20260304.sql

# Restore from compressed backup
gunzip < backup_20260304.sql.gz | mysql -u root -p naalemvdb
```

```bash Docker
# Restore to Docker container
docker exec -i naalemvdb mysql -u root naalemvdb < backup_20260304.sql

# Restore from compressed backup
gunzip < backup_20260304.sql.gz | docker exec -i naalemvdb mysql -u root naalemvdb
```

</CodeGroup>

### Automated Backup Script

```bash
#!/bin/bash
# save as: /home/scripts/backup-naalemv.sh

BACKUP_DIR="/home/backups/naalemv"
DATE=$(date +%Y%m%d_%H%M%S)
FILENAME="naalemvdb_${DATE}.sql.gz"

# Create backup directory if not exists
mkdir -p $BACKUP_DIR

# Create backup
mysqldump -u root -p'your_password' naalemvdb | gzip > "${BACKUP_DIR}/${FILENAME}"

# Keep only last 30 days of backups
find $BACKUP_DIR -type f -name "*.sql.gz" -mtime +30 -delete

echo "Backup completed: ${FILENAME}"
```

<Warning>
Store backup credentials securely. Never hardcode passwords in scripts for production environments. Use MySQL configuration files or environment variables instead.
</Warning>

### Monitor Database Size

```sql
-- Check database size
SELECT 
    table_schema AS 'Database',
    ROUND(SUM(data_length + index_length) / 1024 / 1024, 2) AS 'Size (MB)'
FROM information_schema.tables
WHERE table_schema = 'naalemvdb'
GROUP BY table_schema;

SELECT 
    table_name AS 'Table',
    ROUND(((data_length + index_length) / 1024 / 1024), 2) AS 'Size (MB)'

WHERE table_schema = 'naalemvdb'
ORDER BY (data_length + index_length) DESC;
```

## Common Database Issues

### Issue: Migration Failed

**Symptoms**: Error during `php artisan migrate`

**Solutions**:

```bash
# Check migration status
php artisan migrate:status

# Rollback last migration
php artisan migrate:rollback

# Reset and retry
php artisan migrate:fresh
```

### Issue: Foreign Key Constraint Error

**Symptoms**: Cannot delete/update record due to foreign key constraint

**Solutions**:

```sql
-- Temporarily disable foreign key checks (use with caution)
SET FOREIGN_KEY_CHECKS = 0;
-- Perform operation
SET FOREIGN_KEY_CHECKS = 1;
```

### Issue: Database Connection Failed

**Symptoms**: SQLSTATE[HY000] [2002] Connection refused

**Solutions**:

1. Verify MySQL is running:
   ```bash
   sudo systemctl status mysql
   # or for Docker
   docker ps | grep naalemvdb
   ```

2. Check credentials in `.env`:
   ```bash
   DB_HOST=127.0.0.1
   DB_PORT=3306  # or 3322 for Docker
   DB_DATABASE=naalemvdb
   DB_USERNAME=root
   DB_PASSWORD=
   ```

3. Test connection:
   ```bash
   mysql -h 127.0.0.1 -P 3322 -u root -p
   ```

### Issue: Database Too Large

**Symptoms**: Slow queries, disk space warnings

**Solutions**:

```bash
# Clean old data
php artisan queue:flush
php artisan telescope:prune

# Archive old records
# Implement custom archival logic

# Optimize tables
mysqlcheck -u root -p --optimize naalemvdb
```

### Issue: Charset/Collation Errors

**Symptoms**: Incorrect string value errors

**Solutions**:

```sql
-- Check current charset
SHOW CREATE DATABASE naalemvdb;

-- Convert database to utf8mb4
ALTER DATABASE naalemvdb CHARACTER SET = utf8mb4 COLLATE = utf8mb4_unicode_ci;

-- Convert specific table
ALTER TABLE users CONVERT TO CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

## Security Best Practices

1. **Use Strong Passwords**
   ```sql
   ALTER USER 'root'@'localhost' IDENTIFIED BY 'Strong_P@ssw0rd!';
   ```

2. **Limit User Privileges**
   ```sql
   GRANT SELECT, INSERT, UPDATE, DELETE ON naalemvdb.* TO 'app_user'@'localhost';
   ```

3. **Regular Backups**
   - Schedule daily automated backups
   - Store backups in secure, off-site location
   - Test restore procedures regularly

4. **Enable SSL Connections** (Production)
   ```bash
   # In .env
   DB_SSLMODE=require
   ```

5. **Monitor Access Logs**
   ```sql
   -- Check recent connections
   SELECT * FROM information_schema.processlist;
   ```
